# Research page: highlighting and adding papers

This folder controls the public **Research** page of the site.

The page content lives in [research/index.md](index.md), while the list of publications is generated from data files in [_data](../_data).

---

## 1. Highlight a paper on the Research page

The highlighted paper at the top of the Research page is rendered by this block in [research/index.md](index.md):

```liquid
{% include citation.html lookup="Open collaborative writing with Manubot" style="rich" %}
```

`citation.html` looks up one entry from [_data/citations.yaml](../_data/citations.yaml) using the `lookup` value. It searches in this order:

- First where `citation.id == lookup`
- Otherwise where `citation.title` **contains** `lookup`

### Recommended: use the exact citation id

Each citation generated by the cite process has an `id` field, e.g.:

```yaml
- id: doi:10.18653/v1/2025.acl-short.22
  title: "Combining Domain and Alignment Vectors Provides Better Knowledge-Safety Trade-offs in LLMs"
  # ...
```

To highlight that paper, change the block in [research/index.md](index.md) to:

```liquid
{% include citation.html lookup="doi:10.18653/v1/2025.acl-short.22" style="rich" %}
```

You can use any valid id Manubot understands (for example `doi:...`, `arxiv:...`, `pubmed:...`, etc.), as long as that id exists in [_data/citations.yaml](../_data/citations.yaml).

### Alternative: use part of the title

You can also set `lookup` to a distinctive part of the title:

```liquid
{% include citation.html lookup="GeoCoder - Solving Geometry Problems" style="rich" %}
```

This is less precise (if multiple titles contain the same phrase, only the first match is used), so using the full `id` is preferred.

## 2. Add a new paper (manual sources file)

Citations are **not** edited directly in [_data/citations.yaml](../_data/citations.yaml). That file is generated automatically by the *cite process* from your source data files in [_data](../_data).

For simple manual control, this project uses a single sources file: [_data/sources.yaml](../_data/sources.yaml).

### Step 1 – Add an entry to `_data/sources.yaml`

Open [_data/sources.yaml](../_data/sources.yaml) and add a new item at the end, following the existing examples.

#### Example: add a paper by DOI

```yaml
- id: doi:10.1234/abcd.2025.001
  type: paper
  title: "Example Paper About Language Models"
  authors:
    - Alice Example
    - Bob Example
  publisher: "Example Conference 2025"
  date: "2025-01-01"   # YYYY-MM-DD or YYYY-MM
  link: https://doi.org/10.1234/abcd.2025.001
  image: images/papers/example-lm.jpg   # optional thumbnail used on the site
  tags:
    - language models
    - examples
```

#### Example: add a paper by arXiv id

```yaml
- id: arxiv:2501.01234
  type: preprint
  title: "An Example ArXiv Preprint"
  authors:
    - Jane Doe
    - John Smith
  date: 2025-01-15
  link: https://arxiv.org/abs/2501.01234
  image: images/papers/example-arxiv.jpg
  tags:
    - preprint
    - arxiv
```

Notes:

- **`id`** must be something Manubot understands: `doi:...`, `arxiv:...`, `pubmed:...`, `pmc:...`, etc.
- **`type`** controls the icon shown on the site (see [_data/types.yaml](../_data/types.yaml) for available types).
- `image` is highly recommended so each citation has a thumbnail.
- Any extra fields you add (for example a custom flag for filtering) are passed through unchanged into the generated citation.

## 2. (Optional) Using ORCID and other metasources

This template can also pull publications automatically from services like **ORCID**, **PubMed**, and **Google Scholar**. These are configured in separate files like:

- [`_data/orcid.yaml`](../_data/orcid.yaml)
- `_data/pubmed.yaml`
- `_data/google-scholar.yaml`

Each of these acts as a *metasource* that expands into many individual sources during the cite process. You can still override or correct individual items by adding entries to [_data/sources.yaml](../_data/sources.yaml) with the same `id`.

For example, to fix a publisher name for a source returned via ORCID:

```yaml
- id: doi:10.1234/abcd.2020.999
  publisher: "Correct Publisher Name"
```

During the cite process, entries with the same `id` are merged, and any manually provided fields in `sources.yaml` take precedence.
